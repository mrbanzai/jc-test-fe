Authentication and Security

JobCastle uses HMAC (Hash-based Message Authentication Code) to simultaneously verify both the data integrity and the authenticity of incoming client requests. You can read up on HMAC from this Wikipedia article. Specifically, we use HMAC-SHA1. The mechanism for authenticating with server entails "digitally signing" each of your requests to the API with your provided public and private key. We also lock down incoming requests based on the HTTP referrer, ensuring requests are only coming from a valid client domain (configurable in the admin).

We offer example client libraries written in both PHP and JavaScript.

Summary of Endpoints

Endpoint Description GET POST PUT DELETE  
/states/ Retrieval of all states with available jobs. X view parameters
/states/count/ Retrieval of the total number of states with available jobs. X view parameters
/cities/ Retrieval of all cities with available jobs. X view parameters
/cities/count/ Retrieval of the total number of cities with available jobs. X view parameters
/categories/ Retrieval of all categories with available jobs. X view parameters
/categories/count/ Retrieval of the total number of categories with available jobs. X view parameters
/schedules/ Retrieval of all job schedules with available jobs. X view parameters
/schedules/count/ Retrieval of the total number of job schedules with available jobs. X view parameters
/jobs/ Retrieval of all jobs, including full details. X view parameters
/jobs/id/ Retrieval of details for a specific job by id. X view parameters
/jobs/uristub/ Retrieval of details for a specific job by uristub. X view parameters
/jobs/count/ Retrieval of a count of all jobs matching a distinct set of criteria. X view parameters
/jobs/titles/ Retrieval of all job titles. X view parameters
/jobs/radius/ Retrieval of all jobs within a given radius of a zipcode. X view parameters
/jobs/search/ Retrieval of all jobs matching a distinct set of criteria. X view parameters

/jobs/count/

All search filters are joined by an AND operation; they are mutually inclusive on AND. If you would like to search for multiple city/state combinations, you should use the location parameter. It overrides both city and state (as it represents a combination of the two).

Parameter(s) Required? Notes Accepted Values
state No A search filter for selecting jobs within a given state. string|array
Expects a string or array of two digit state codes or full state names.
city No A search filter for selecting jobs with a given city. string|array
Expects a string or array of cities.
location No A search filter for selecting jobs with a given location (city/state). string|array
Expects a string or array of locations.
category No A search filter for selecting jobs with a given city. string|array
Expects a string or array of categories.
schedule No A search filter for selecting jobs with a given job schedule. string|array
Expects a string or array of job schedules.

Error Codes and Handling

Every response contains a parameter status_code. These status codes can be used to determine if an error has occurred in the application or not. Below is a listing of all status codes and their associated meaning. If you receive a status_code other than 200 or 304, you should also check the returned errors parameter as it may contain additional information regarding problems with your request.

Error Codes

Status Code Type Reason
200 SUCCESS Your request was successful.
304 UNMODIFIED Your request was successful. Data has not changed.
400 BADREQUEST The request could not be understood by the server due to missing or malformed parameters.
401 UNAUTHORIZED Unauthorized access. The request was not signed properly or your public/private key pair is incorrect for your domain.
403 FORBIDDEN The server understood the request, but is refusing to fulfill it. You may not have access to the API call.
404 NOTFOUND No API method exists matching the URL specified.
405 INVALID The request method was not valid.
420 CHILL You have exceeded your rate limit.
500 INTERNALERROR The server encountered an unexpected condition which prevented it from fulfilling the request.
402 OVERLOAD The server is currently unable to handle the request due to a temporary overloading or maintenance of the server.
